1. 시스템 개요

KMDB(한국영화데이터베이스)에서 수집한 19,354편(1980~2026년)의 영화를 499차원 하이브리드 벡터로 변환하고, 코사인 유사도를 기반으로 가장 유사한 영화 Top-20을 추천하는 콘텐츠 기반 필터링 시스템이다.

2. 추천 원리

콘텐츠 기반 필터링의 직관

"영화 A와 영화 B가 비슷한 장르, 비슷한 키워드, 비슷한 줄거리를 가지면, A를 좋아하는 사람은 B도 좋아할 것이다."

이 직관을 수학적으로 구현하려면 두 가지 문제를 해결해야 한다:

  1. 영화를 어떻게 숫자로 표현할 것인가? → 임베딩(Embedding): 영화의 속성을 고차원 벡터로 변환
  2. "비슷하다"를 어떻게 측정할 것인가? → 코사인 유사도: 두 벡터의 방향 일치도를 -1~1로 측정

코사인 유사도의 의미

cos(A, B) = (A . B) / (||A|| x ||B||) = cos(theta) A . B = 두 벡터의 내적 (각 차원을 곱해서 합산) ||A|| = 벡터 A의 크기 (L2 노름) theta = 두 벡터 사이의 각도
값의미예시
1.0완전히 같은 방향 (동일한 특성)같은 장르, 같은 키워드, 같은 줄거리
0.7~0.9매우 유사같은 장르 + 유사 줄거리
0.3~0.7부분적 유사일부 장르 공통, 줄거리 테마 유사
0.0무관공통점 없음
-1.0정반대이론상. 원-핫/바이너리에서는 거의 발생하지 않음
왜 유클리드 거리가 아닌 코사인 유사도를 사용하는가?
유클리드 거리는 벡터의 크기에 영향을 받지만, 코사인 유사도는 벡터의 방향만 비교한다. 키워드가 많은 영화(벡터 크기 큼)와 적은 영화(벡터 크기 작음)를 공정하게 비교하려면 방향 비교가 적합하다. L2 정규화를 각 그룹에 적용하는 이유도 동일한 맥락이다.

인코더 모델 vs 생성형 LLM: sentence-transformers는 텍스트를 고정 차원 벡터로 변환하는 인코더이다. 추천 로직에 LLM은 전혀 사용되지 않으며, 오직 코사인 유사도만으로 추천 순위를 결정한다.

3.2 L2 정규화 (그룹별)

문제: 텍스트 그룹(384D)이 장르(30D)보다 12.8배 더 많은 차원을 차지하므로, 단순 연결하면 텍스트가 유사도를 지배한다.

해법: 각 그룹을 독립적으로 L2 정규화하여 크기를 1로 통일한 후, 가중치를 곱한다.

g_norm = g / ||g||2 # 장르 30D: 크기를 1로 정규화 k_norm = k / ||k||2 # 키워드 80D: 크기를 1로 정규화 n_norm = n / ||n||2 # 수치 5D: 크기를 1로 정규화 t_norm = t / ||t||2 # 텍스트 384D: 크기를 1로 정규화 V = [g_norm x Wg | k_norm x Wk | n_norm x Wn | t_norm x Wt] = [30D x 1.0 | 80D x 1.0 | 5D x 0.5 | 384D x 1.5] = 499D 최종 벡터
L2 정규화 후 가중치의 효과: 정규화로 각 그룹의 L2 노름이 1이 되므로, 가중치 W는 해당 그룹이 최종 유사도에 기여하는 비율을 직접 조절한다. Wt=1.5는 텍스트 정보가 장르(1.0)보다 1.5배 더 영향을 준다는 의미이다.

3.3 두 종류의 벡터

벡터용도변환사용처
raw_vectors 추천 (유사도 계산) 그룹별 L2 정규화 + 가중치 곱 코사인 유사도, 그룹별 기여도 분해
scaled_vectors 군집화, 차원 축소 raw_vector → StandardScaler (평균=0, 분산=1) KMeans, PCA/t-SNE
왜 다른 벡터를 사용하는가?
추천: raw 벡터는 가중치가 직접 반영되어 있어, 코사인 유사도 계산 시 사용자가 설정한 그룹별 중요도가 정확히 반영된다.
군집화/PCA: StandardScaler로 각 차원의 스케일을 통일해야 KMeans의 유클리드 거리와 PCA의 분산 계산이 편향 없이 작동한다.

3.4 텍스트 임베딩 (sentence-transformers)

paraphrase-multilingual-MiniLM-L12-v2 모델은 한국어를 포함한 50+ 언어의 텍스트를 384차원 밀집 벡터로 변환한다. 줄거리의 의미적 유사성을 포착한다.

4. 처리 흐름 (6단계 파이프라인)

1

데이터 로드 (data_loader.py)

554개 JSON 파일을 순회하며 19,354편 영화를 파싱한다. KMDB 원본 66개 장르를 30개 표준 카테고리로 정규화하고, 빈도 상위 80개 키워드를 매칭하며, 런타임/연도/출연진 등 5개 파생 수치를 계산한다. 결과를 kmdb_processed.pkl로 캐싱하여 재파싱을 방지한다.

2

임베딩 생성 (embedding.py)

각 영화를 499D 하이브리드 벡터로 변환한다. 메타데이터 벡터(장르+키워드+수치 = 115D)를 직접 구성하고, 줄거리 텍스트를 sentence-transformers로 384D 벡터로 인코딩한다. 4개 그룹을 L2 정규화 후 가중치를 곱하고 연결하여 raw 벡터를 생성한다. 이후 전체에 StandardScaler를 적용하여 scaled 벡터도 생성한다.

3

군집화 (clustering.py)

scaled 벡터에 KMeans(k=12)를 적용하여 영화를 12개 클러스터로 분류한다. GPU CUDA 사용 시 PyTorch 텐서 연산으로 가속한다. 각 클러스터의 주요 장르/키워드 통계를 산출한다.

4

차원 축소 (reduction.py)

499D scaled 벡터를 PCA 또는 t-SNE로 3D/2D 좌표로 축소한다. PCA의 경우 SVD(특이값 분해)를 사용하며, 각 축의 설명 분산비와 기여 피처를 해석한다. 시각화(산점도)에 사용된다.

5

추천 (recommender.py)

테스트 영화의 raw 벡터와 학습 영화의 raw 벡터 사이에 코사인 유사도를 계산하여 Top-20을 선정한다. 각 추천 결과에 대해 장르/키워드/수치/텍스트 그룹별 유사도를 분해하여 "왜 이 영화가 추천되었는가"를 설명한다.

6

평가 (evaluator.py)

추천 결과를 정량적(유사도, 장르정밀도, 키워드정밀도, 다양성, 텍스트일관성)으로 측정하고, 정성적(장르분포, 연도범위, 국가다양성, Top1 직관성)으로 분석한다. 임계값 기반으로 적합/부적합을 판정한다.

5. 수식 체계

5.1 벡터 생성

L2 정규화

x_norm = x / ||x||2 ||x||2 = sqrt(x12 + x22 + ... + xn2) # 영벡터(||x|| < 10^-10)인 경우 ||x|| = 1로 대체하여 0/0 방지

가중 벡터 연결 (Weighted Concatenation)

V = concat( L2norm(g) x Wg, L2norm(k) x Wk, L2norm(n) x Wn, L2norm(t) x Wt ) # 기본 가중치: W_g=1.0, W_k=1.0, W_n=0.5, W_t=1.5 # V의 차원: 30 + 80 + 5 + 384 = 499

5.2 표준화 (StandardScaler)

muj = (1/N) x SUM(Xij) # j번째 차원의 평균 sigmaj = sqrt( (1/N) x SUM((Xij - muj)2) ) # j번째 차원의 표준편차 X_scaledij = (Xij - muj) / sigmaj # 평균=0, 분산=1로 변환 # sigma_j < 10^-10 이면 1.0으로 대체 (상수 차원)

5.3 코사인 유사도

기본 수식

cos(A, B) = (A · B) / (||A|| × ||B||) # 내적 전개: A · B = SUM(Ai × Bi) for i = 1..499 # GPU 배치 연산 (M개 쿼리, N개 학습): Q_n = Q / ||Q|| (행별 L2 정규화, M x 499) T_n = T / ||T|| (행별 L2 정규화, N x 499) S = Q_n × T_nT (행렬곱, M x N 유사도 행렬) # 정규화된 벡터의 내적 = 코사인 유사도: # (A / ||A||) · (B / ||B||) = A · B / (||A|| × ||B||)

그룹별 유사도 분해

sim_genre(A, B) = cos(gA, gB) # 장르 그룹만의 유사도 sim_keyword(A, B) = cos(kA, kB) # 키워드 그룹만의 유사도 sim_numeric(A, B) = cos(nA, nB) # 수치 그룹만의 유사도 sim_text(A, B) = cos(tA, tB) # 텍스트 그룹만의 유사도 sim_total(A, B) = cos(VA, VB) # 전체 499D 유사도 # 주의: sim_total != sim_genre + sim_keyword + sim_numeric + sim_text # 전체 유사도는 가중 연결된 벡터의 코사인으로, 단순 합이 아님

전체 유사도의 그룹별 전개 (핵심 수식)

499D 벡터 V 는 L2 정규화된 4개 그룹에 가중치를 곱한 연결이다:

VA = [ (gA/||gA||)×Wg | (kA/||kA||)×Wk | (nA/||nA||)×Wn | (tA/||tA||)×Wt ]

내적 VA · VB 를 그룹별로 전개하면:

VA · VB = Wg2 × cosgenre(A,B) + Wk2 × coskeyword(A,B) + Wn2 × cosnumeric(A,B) + Wt2 × costext(A,B) # 각 그룹이 L2 정규화되어 노름=1이므로: ||V||2 = Wg2 + Wk2 + Wn2 + Wt2

따라서 최종 코사인 유사도는 가중치 제곱(W²)을 비율로 한 그룹별 코사인 유사도의 가중 평균이 된다:

cos(A, B) = Wg2 × cosg + Wk2 × cosk + Wn2 × cosn + Wt2 × costWg2 + Wk2 + Wn2 + Wt2

기본 가중치에서의 실제 기여 비율

그룹WW²기여 비율의미
장르1.01.0022.2%30D 원-핫 인코딩
키워드1.01.0022.2%80D 바이너리 인코딩
수치0.50.255.6%5D 파생 수치
텍스트1.52.2550.0%384D 줄거리 임베딩
합계--4.50100%
핵심 해석: 텍스트(줄거리) 유사도가 전체의 절반(50%)을 차지하고, 장르와 키워드가 각 22%씩, 수치 특성이 약 6%를 차지한다. 가중치 W를 변경하면 W² 비율이 바뀌어 각 그룹의 기여 비율이 달라진다.

5.4 KMeans 군집화

입력: X_scaled (N x 499), k=12 초기화: centroidc = X_scaled[random_index[c]] for c = 0..11 반복 (max 300회): (1) distic = ||Xi - centroidc||2 # 유클리드 거리 (2) labeli = argminc(distic) # 가장 가까운 중심 배정 (3) centroidc = mean(X[label==c]) # 중심 갱신 (4) if max|centroid_old - centroid_new| < 10-6: 종료

5.5 PCA 차원 축소

입력: X_scaled (N x 499) (1) 중심화: X_c = X_scaled - mean(X_scaled) (2) 특이값 분해 (SVD): X_c = U x S x VT # U: N x N (좌특이벡터) # S: min(N,499) 특이값 (대각행렬) # V^T: 499 x 499 (우특이벡터, 주성분 방향) (3) 투영 (3D): X_reduced = X_c x VT[:3]T (N x 3) (4) 설명 분산비: ratioi = Si2 / SUM(Sj2) # PC1이 전체 분산의 몇 %를 설명하는지

5.6 Spearman 순위 상관계수

rho = 1 - (6 x SUM(di2)) / (n x (n2 - 1)) # d_i = rank_baseline(movie_i) - rank_combo(movie_i) # n = 공통 영화 수 # 해석: # rho = 1.0 : 순위 완전 동일 # rho = 0.0 : 순위 무관 # rho = -1.0 : 순위 완전 역전

4-1. 한글 음절 → 자모 분해

유니코드 한글 음절(U+AC00~U+D7A3)을 초성/중성/종성 인덱스로 분해한다:

offset=code(음절) - 0xAC00
종성=offset mod 28
중성=⌊offset / 28⌋ mod 21
초성=⌊offset / (28 × 21)⌋

4-2. Levenshtein 편집 거리 (DP)

d(i, j)  =  min { d(i-1, j) + 1 (삭제)
d(i, j-1) + 1 (삽입)
d(i-1, j-1) + cost (치환: 같으면 0, 다르면 1)

edit_distance(A, B) = d(|SA|, |SB|) # 조기 종료: 행의 최솟값이 max_dist를 초과하면 즉시 중단 # 메모리 절약: 2행 DP (prev, curr)로 O(min(|A|,|B|)) 공간 사용

9.4 기여도 산출 방식 (Contribution Importance)

각 파라미터(장르/키워드/수치/텍스트)가 추천 결과에 미치는 영향력을 Marginal Range 방식으로 정량화한다.

기여도 산출 알고리즘 파라미터 P ∈ {Wg, Wk, Wn, Wt} 에 대해: 1) 수준별 평균 유사도 계산: μP=하 = mean(avg_similarity | P=0.5) # P가 "하"인 모든 조합(27개)의 평균 유사도 μP=중 = mean(avg_similarity | P=1.0) # P가 "중"인 모든 조합(27개)의 평균 유사도 μP=상 = mean(avg_similarity | P=1.5) # P가 "상"인 모든 조합(27개)의 평균 유사도 2) 중요도(Importance) = max(μ하, μ중, μ상) - min(μ하, μ중, μ상) # 수준별 평균의 최대-최소 범위. 범위가 클수록 영향력이 큼 3) 최적 수준(Best Level) = argmax(μ하, μ중, μ상) # 평균 유사도가 가장 높은 수준 4) 4개 파라미터를 중요도 내림차순으로 정렬하여 기여도 순위 결정

기여도 해석

기여도 크기의미예시
높음 (예: 0.05+)해당 가중치 변경이 추천 결과를 크게 바꿈텍스트 가중치: 줄거리 기반 유사도가 지배적
중간 (예: 0.01~0.05)의미 있는 영향이 있으나 지배적이지 않음장르 가중치: 장르 일치가 부분적으로 기여
낮음 (예: <0.01)해당 가중치 변경이 추천에 미미한 영향수치 가중치: 5D로 차원이 작아 영향 제한적

9.5 해석 가이드

높은 겹침률 + 높은 Spearman ρ

가중치 변경이 추천에 미치는 영향이 적다. 해당 그룹의 정보가 다른 그룹과 중복되거나, 추천에 결정적이지 않다.

낮은 겹침률 + 낮은 Spearman ρ

가중치 변경이 추천을 크게 바꾼다. 해당 그룹이 추천 결과에 강한 영향력을 가진다.

기여도 1위 = 텍스트

384D 줄거리 임베딩이 유사도를 지배. 장르나 키워드보다 줄거리의 의미적 유사성이 추천을 결정.

기여도 1위 = 장르

장르 일치가 추천의 핵심. 같은 장르 내에서 세부 차이를 텍스트/키워드가 보완.

10. 가중치 최적화 (시뮬레이티드 어닐링)

10.1 개요

4개 가중치(Wg, Wk, Wn, Wt)의 최적 조합을 시뮬레이티드 어닐링(Simulated Annealing)으로 탐색한다. 이 메타휴리스틱 최적화는 격자 탐색(Grid Search)보다 넓은 연속 공간을 효율적으로 탐색하며, 지역 최적해(local optimum)에 빠지지 않도록 확률적 이동을 허용한다.

10.2 복합 목적함수 (Composite Score)

S(w) = 0.35 · avg_similarity + 0.30 · genre_precision + 0.25 · text_coherence - 0.10 · |diversity - 0.3| 각 항의 역할: avg_similarity (0.35) : 추천 영화의 평균 코사인 유사도 # 높을수록 유사한 영화 추천 genre_precision (0.30) : Top-20 중 쿼리와 장르 공유 비율 # 높을수록 장르 관련성 높음 text_coherence (0.25) : 추천 영화 줄거리 간 의미적 일관성 # 높을수록 주제 일관 diversity_penalty(0.10) : |diversity - 0.3| # 0.3이 이상적 다양성, 편차 시 감점

항들의 관계

항계수방향다른 항과의 관계
avg_similarity0.35 (최대)최대화높이면 diversity가 낮아지는 경향 (trade-off)
genre_precision0.30최대화장르 가중치 증가 시 개선, 텍스트 가중치와 독립
text_coherence0.25최대화텍스트 가중치 증가 시 개선, 장르와 부분 상관
diversity_penalty0.10최소화similarity와 역상관: 유사도 높이면 다양성 감소
설계 의도: 유사도(0.35)와 장르 정밀도(0.30)를 합쳐 0.65로 "정확성"을 주로 추구하되, 텍스트 일관성(0.25)으로 의미적 품질을 보장하고, 다양성 패널티(0.10)로 단조로운 추천을 방지한다. 다양성 목표값 0.3은 "적당히 다양하되 주제에서 벗어나지 않는" 균형점이다.

10.3 시뮬레이티드 어닐링 알고리즘

알고리즘 (optimizer.py) 초기 설정: wcurrent = 사용자 지정 초기 가중치 (슬라이더 값) T0 = 1.0 # 초기 온도 α = 0.92 # 냉각률 (기하 냉각) step = 0.2 # 이웃 탐색 반경 patience = 10 # 개선 없이 허용하는 반복 수 반복 (최대 50회): 1) 이웃 생성: key = 무작위 선택(genre, keyword, numeric, text) δ = Uniform(-step, +step) wneighbor[key] = clip(wcurrent[key] + δ, 0.1, 3.0) 2) 평가: Sneighbor = S(wneighbor) # 복합 목적함수 계산 3) 수락 판정: ΔS = Sneighbor - Scurrent if ΔS > 0: accept # 항상 개선 수락 else: P(accept) = exp(ΔS / T) # 확률적 악화 수락 4) 온도 감소: T ← α · T # T_n = 0.92^n (기하급수 냉각) 5) 조기 종료: patience 회 연속 미개선 시 중단

10.4 신뢰도 (Confidence) 산출

Confidence = 0.4 · stability + 0.4 · reproducibility + 0.2 · convergence stability = max(0, 1 - std(tail_25%_scores) · 10) # 후반부 점수 안정성 reproducibility = min(1, count(|score - best| < 0.05·best) / (n·0.3)) # 최적 근처 도달 빈도 convergence = min(1, n / 20) # 충분한 탐색 여부
요소가중치의미
stability0.4탐색 후반부 점수의 표준편차가 작을수록 안정적으로 수렴
reproducibility0.4최적값 근처(5% 이내)에 자주 도달할수록 재현성 높음
convergence0.2최소 20회 이상 탐색했으면 충분한 탐색으로 간주

10.5 정확도 (Accuracy) 산출

Accuracy = (통과한 조건 수) / 4 조건: 1) avg_similarity ≥ 0.5 # THRESHOLD_AVG_SIMILARITY 2) genre_precision ≥ 0.3 # THRESHOLD_GENRE_PRECISION 3) text_coherence ≥ 0.3 # 의미적 일관성 최소 기준 4) 0.15 ≤ diversity ≤ 0.6 # 다양성 허용 범위

10.6 저장 및 반영 조건

시뮬레이션 결과는 신뢰도 ≥ 50% 이고 정확도 ≥ 50% 일 때만 "저장 및 반영" 가능. 이 조건을 충족하면 슬라이더에 최적 가중치가 적용되고, 파일로 영속화된다.

11. 가중치 영속화

11.1 개요

시뮬레이션 또는 수동으로 설정한 가중치를 JSON 파일(data/saved_weights.json)에 저장하여, 대시보드를 재시작해도 이전 설정이 유지되도록 한다.

11.2 저장 구조

{ "weights": { "genre": 1.2, "keyword": 0.8, "numeric": 0.7, "text": 1.6 }, "metadata": { "saved_at": "2026-03-26T14:30:00", "source": "simulation", "confidence": 0.72, "accuracy": 0.75, "composite_score": 0.6234 } }

11.3 로드 우선순위

get_effective_weights(): 1) data/saved_weights.json 존재 & 유효 → 저장된 가중치 사용 2) 파일 없음 또는 파싱 실패 → config.py 기본값 사용 (WEIGHT_GENRE=1.0, WEIGHT_KEYWORD=1.0, WEIGHT_NUMERIC=0.5, WEIGHT_TEXT=1.5)

11.4 대시보드 연동

동작함수효과

12.3 정리

강점: 4개 그룹을 독립적으로 L2 정규화한 후 가중치로 통합하는 설계는, 이질적인 피처(이산/연속/밀집)를 공정하게 결합하면서도 각 그룹의 영향력을 투명하게 조절할 수 있게 한다. GPU 가속과 캐싱으로 19,000편 규모에서도 실용적인 속도를 유지한다.
근본적 한계: 사용자 행동 데이터 없이 콘텐츠 속성만으로 추천하므로, "이 영화를 좋아한 사람은 이것도 좋아합니다" 같은 개인화가 불가능하다. 이는 콘텐츠 기반 필터링의 본질적 한계이며, 협업 필터링과의 결합이 필요하다.
